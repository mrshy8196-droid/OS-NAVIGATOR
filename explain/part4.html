The MMU is a hardware component that handles the mapping between Logical Addresses (generated by the CPU) and Physical Addresses (actual locations in RAM).

2. Swapping Strategy

Swapping allows processes to be moved between main memory and a backing store (disk) to accommodate more programs.

  • Swap Out: RAM → Disk
  • Swap In: Disk → RAM

Advanced Virtual Memory & Paging

1. Page Table Structures

Hierarchical Paging: Breaking the page table into multiple levels (e.g., 2-level paging) to avoid storing huge contiguous tables in memory.

Outer Page Table → Page of Page Table → Memory Frame

Hashed Page Table: Common for address spaces > 32 bits. Virtual page numbers are hashed into a table entry.

Inverted Page Table: One entry for each real page (frame) in memory, reducing table size significantly.

2. Segmentation with Paging

Segmentation supports the user view of memory (Main, Objects, Stack). By combining it with Paging, we divide segments into equal-sized pages, providing better memory allocation and protection.

3. Page Replacement Algorithms

When a Page Fault occurs and memory is full, we must victimize a page:

  • Optimal: Replace page that won't be used for the longest time.
  • FIFO: First-In, First-Out (Oldest page is replaced).
  • LRU: Least Recently Used (Replaces page not used for the longest backward distance).
  • LFU: Least Frequently Used (Based on reference count).

File Access Methods

Direct Access: Allows reading or writing records rapidly in no particular order.
